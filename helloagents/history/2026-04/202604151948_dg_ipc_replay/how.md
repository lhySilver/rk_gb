# 技术设计: dg_ipc 分支整理回放

## 技术方案
### 核心技术
- Git worktree 隔离工作区
- 在新分支上顺序吸收协议分支独有提交与 `dg_ipc` 提交
- 使用路径级回退清理明显的构建残留

### 实现要点
- 新分支从当前本地 `feature/gb-zero-config-macro-switch-20260326` 起步，保留本地未推送提交。
- 先吸收远端协议分支独有提交 `9608fec`，完成协议线合流。
- 对 `0fb527e` 使用无提交方式回放，再回退明显垃圾路径，最后生成干净的 IPC 回放提交。

## 架构设计
本次不改变整体模块架构，仅调整分支组合方式与 IPC 适配代码落点。

## 安全与性能
- **安全:** 不执行破坏性重置，不覆盖用户当前工作区，不删除未确认的本地提交。
- **性能:** 仅清理构建目录和调试垃圾，不引入额外运行时逻辑开销。

## 测试与部署
- **测试:** 至少执行 Git 状态检查、提交检查，以及项目可用的最小构建验证。
- **部署:** 结果落在独立分支 `feature/dg_ipc_replay_20260415`，不直接改用户当前分支。
