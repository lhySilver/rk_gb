# 技术设计: issue47 平台 SDK 冗余文件清理

## 技术方案

### 核心技术
- Git / CMake
- 仓库级 `#include` 引用分析
- `git diff --check` 与交叉编译构建验证

### 实现要点
- 以 `App/CMakeLists.txt` 和 `App/Protocol/gb28181/sdk_port/CMakeLists.txt` 中显式列出的 `platform_sdk_port` 源文件作为构建根节点
- 扫描仓库内所有源码的 `#include`，把非目标目录对 `CommonFile` / `CommonLibSrc` 的直接引用作为可达根节点
- 从上述根节点递归展开目标目录内部的 include 关系，得到当前分支真实可达文件集合
- 对可达图之外的文件做人工抽样复核，确认不存在 BOM 导致的首行 include 漏判等假阳性后再删除
- 删除动作只针对仓库内不可达文件，不调整当前构建脚本中的 include 目录和显式源码列表

## 安全与性能
- **安全:** 不触碰协议逻辑和运行时配置，只删除仓库内未接入当前构建的冗余文件
- **性能:** 直接收敛第三方历史遗留目录，降低后续搜索、审查和误引用成本

## 测试与部署
- **测试:** 运行 `git diff --check`，并按现有交叉编译命令重新配置 / 构建主工程验证本次清理未破坏现有编译链
- **部署:** 无额外部署动作，保持现有打包与运行方式不变
